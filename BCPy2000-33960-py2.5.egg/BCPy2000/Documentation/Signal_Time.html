Signal Time

By signal time we mean the nominal time that the signal-processing code believes has elapsed since the start of a run, computed according to the SamplingRate parameter and the number of samples processed. Signal time may differ from real timeâ€”

  • if the signal is being artificially generated (or played back from a recorded file) by a BCPy2000-implemented signal source module, and the WarpFactor parameter is set to something other than 1.0;
  • to the extent that the signal acquisition hardware and driver fail to realize the requested SamplingRate exactly;
  • if packets have been skipped or delayed, for example due to processor overloading.

If the BCPy2000 application module's ShowSignalTime parameter is set to True, then the signal time and packet number are displayed in the bottom right corner of the VisionEgg window.
