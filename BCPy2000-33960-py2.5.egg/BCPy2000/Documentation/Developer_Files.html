Developer Files

The developer files are the files written and maintained by you:

  • BciSource.py - the default name for your python implementation of the signal-generation module
  • BciSignalProcessing.py - the default name for your python implementation of the signal-processing module
  • BciApplication.py - the default name for your python implementation of the application module
  • any other resources (stimulus images, sounds, etc)

By default, the framework looks for these in the working directory, which by default is ..\python relative to the BCI2000 installation directory. However, this path, as well as the names of the developer files, can be changed using command-line options passed to the foundation binaries.

When a BCPy2000 module is launched, it adds the working directory to its sys.path and also changes to this directory using os.chdir. Any relative paths to resources files (for example, sound and image files that you might want to load) will therefore be computed relative to here, from the point of view of your python code. (Note that if you keep the default working-directory setting, any relative path starting with .. will reach the same destination whether one starts from the working directory or from the installation directory.)
