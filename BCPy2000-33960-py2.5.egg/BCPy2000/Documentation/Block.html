Block

Definition: block

A concentrated period of experimental task performance typically lasting 5â€“15 minutes (perhaps divided up into as few as 10 trials or as many as 200, depending on their length) after which the subject might wish to take a break for a minute, or a few minutes.

Usually this is synonymous with run, although in BCPy2000 application modules it is possible to specify multiple BlocksPerRun and to use the self.design API function to define an interblock phase. If so, the CurrentBlock state keeps track of the block number within the run.

Note that the official BCI2000 documentation may sometimes use the word "block" as a shorthand for "sample block" which means something different. To avoid confusion, we use the term "packet" (or "signal packet") to denote batches of samples.
