Signal Processing Binary

The signal processing binary resides in the BCI2000 installation directory and is called PythonSignalProcessing.exe.

The abbreviation "PythonSig" is used as a prefix for all the module's command-line options, as the name of the main BCI2000 filter registered by the module, and as the name of the config tab in which some of the modules parameters can be set.

The signal processing binary does the following:

  1. starts an embedded python engine
  2. opens a console window (unless the command-line options specify a log file and explicitly disable the shell)
  3. imports symbols from the framework module BCPy2000.GenericSignalProcessing
  4. changes to the desired working directory and adds it to python's sys.path
  5. runs the developer's signal-processing file (by default, called BciSignalProcessing.py)
  6. creates an instance of the developer-defined class BciSignalProcessing (referred to as self)
  7. starts an interactive IPython shell if desired
  8. acts as an intermediary between self's methods and (via TCP/IP connections) the other modules of the BCI2000.
