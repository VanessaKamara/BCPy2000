Foundation

By foundation we mean a collection of BCI2000 binary executables. Usually they are all together in one directory, which BCI2000 calls the installation directory. The installation directory is conventionally named prog , and for binaries it is the starting point with respect to which relative file paths are expressed (for your python code, the starting point is the working directory).

The executables are:

  • the operator;
  • any other non-python-related BCI2000 modules you wish to use;
  • and then one Python-related binary for each type of core BCI2000 module:
    • PythonSource.exe
    • PythonSignalProcessing.exe
    • PythonApplication.exe

The binaries' behaviour is influenced by various command-line options.
