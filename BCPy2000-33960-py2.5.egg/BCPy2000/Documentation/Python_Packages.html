Python Packages

The table below shows download information on BCPy2000's dependencies. It is provided in order to be transparent, and to help keep things organized at our end, but you do not have to download everything separately. If you want, you can just go straight to the download page and download the whole lot pre-packaged.

Since we are dependent on Windows, the version numbers in the table are direct links to the win32 binary installers. Once downloaded, each package usually only takes a few seconds to install. Make sure, however, that you install them in the order listed.

A complete update of the information was last made on 2009-09-12. Some packages to which we link are not the very latest versions---either for specific reasons given in the table, or simply because we have not yet tested later versions for compatibility with BCPy2000.

A * next to a download link denotes a pure-Python package which, like the BCPy2000 framework package itself, should be equally installable under any Python version 2.x. All other links are to Python-2.5.4-specific binary installers: this is the Python version we recommend, since it seems to be the most popular and widely-supported version among scientists. For Python 2.4, you would additionally need the ctypes package. For Python 2.6, you may find that some dependencies' binary installers are not yet available, and you would need to compile those packages yourself.

If you ever want or need to install things from source, pure-Python packages can be installed with the command setup.py install whereas the rest require compilation of C or C++ sources (see the page on developing Python for Windows).

Finally, if you're a fan of easy_install, the binary files linked below will also work as inputs to the easy_install program, which will simply extract eggs from them. Exceptions are the NumPy and SciPy "superpacks", which are meta-packages that have to be unzipped first, using a program like 7-Zip: the result is three separate .exe files, of which the one marked sse3 seems appropriate for most CPUs built in the last few years.
