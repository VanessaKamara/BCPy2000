Framework

The framework is a collection of python files written and maintained by us. They contain the necessary glue between the binary executables of the foundation and the python code that you write in your developer files. They also define various API functions that you can call in your implementation.

The framework consists of the following files. First, each of the executables requires its corresponding python module:

  • BCPy2000.GenericSource
  • BCPy2000.GenericSignalProcessing
  • BCPy2000.GenericApplication

...all of which depend on

  • BCPy2000.Generic

... as well as a couple of more general-purpose dependencies:

  • Diagnostics.py
  • EmbeddedPythonConsole.py
  • PrecisionTiming.py

By default, the foundation executables expect these files to be installed as a python package (under Lib/site-packages in your python distribution). The BCPy2000 installer should do this for you. For instructions on updating the package to subsequent versions, see the updating page.
