AppTools

This package is part of the evolving set of optional higher-level tools. The developer interfaces to some of the undocumented AppTools may change slightly in future.

The package consists of:

  • AppTools.Boxes (documented) implements the box and point classes which make it convenient to create, manipulate, and map between rectangular coordinate frames.
  • AppTools.ConfusionMatrices (undocumented) will provide a confusion_matrix class for displaying classification performance.
  • AppTools.Displays (documented) provides an interface for finding out how many screens are attached and what their coordinates are. Requires the third-party win32api python package.
  • AppTools.Meters (undocumented) will provide classes for visualizing input values as bars whose lengths and text annotations change in real time.
  • AppTools.ParallelPort (documented) provides a simple interface to the computer's parallel ports, via the inpout32.dll which must be installed in the system directory.
  • AppTools.Shapes (undocumented) provides a PolygonTexture function, which wraps around VisionEgg and PIL calls to make polygon stimuli, as in the demo.
  • AppTools.StateMonitors (undocumented) implements stimulus objects that display the values of state variables, or (by setting the func attribute) arbitrary other quantities. The display is automatically updated on every video frame. State monitors can be seen in action in the demo if the ShowSignalTime parameter is turned on.
  • AppTools.StimulusQueue (undocumented) allows pygame or WavTools sound stimuli to be queued, i.e. to be scheduled to be played automatically one after another (at least, to within the precision of one video frame).
