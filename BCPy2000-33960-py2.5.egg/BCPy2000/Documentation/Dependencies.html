Dependencies

Windows

During development of our current release, BCI2000 itself has only officially supported Windows. Therefore, we can also only support (32-bit) Windows. So far, we have only been able to perform extensive testing of our system on Windows XP Professional SP2 and SP3.

However, a large proportion of the BCI2000 source-code is already cross-platform compatible, and we have had some success in compiling and running the BCPy2000 modules on other platforms.

Note that, in any case, a large proportion of commercial EEG signal acquisition systems will remain Windows-dependent for the foreseeable future.

BCI2000

To use BCPy2000, you need the BCI2000 binaries. The bare minimum for a BCPy2000 demo is as follows:

  • the central BCI2000 operator, operat.exe, from the main BCI2000 distribution;
  • our PythonSource.exe ;
  • our PythonSignalProcessing.exe ;
  • our PythonApplication.exe.

In addition, to use the framework for real experiments, you will need whichever signal-acquisition binary from the main BCI2000 distribution is appropriate for your EEG hardware.

The BCI2000 source code and binaries are available for free to researchers. Note, however, that the BCI2000 project's support system is such that you will need to register and sign an agreement before you can access the full distribution. See the BCI2000 website for details.

The parts written by us, i.e. the python-specific C++ code for our three binaries, the framework that makes the bridge between BCI2000 and Python, and the optional Python tools, are open-source. Our sources are also integrated into the official BCI2000 source distribution, under src/contrib/BCPy2000/foundation and can be built from there.

Python

Before you can use BCPy2000, you must install python and a number of third-party python packages. BCPy2000 admittedly has a lot of dependencies, but all of them are free open-source packages, and most of them should be directly useful to you as developer, just as they were for us in building the framework.

Our framework itself is also a Python package. The files are included in the BCI2000 source distribution under src/contrib/BCPy2000/framework and can be installed from there.
