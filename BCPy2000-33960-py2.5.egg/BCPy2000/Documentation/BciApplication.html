BciApplication

BciApplication is the name of a class, implemented by you in one of the developer files. The filename, by default, is also BciApplication.py but this can be changed by specifying the command-line option --PythonAppClassFile to the application binary.

The BciApplication class is instantiated when the BCI2000 starts, and if the application module's shell is running, you can address this instance from the shell prompt as self.

Your BciApplication must be a subclass of BciGenericApplication, which is defined in the framework module BCPy2000.GenericApplication. Your implementation should not define __init__ and __del__ methods, but rather use the appropriate hooks named according to the BCI2000 conventions.

The API for applications is rather more extensive than for the other BCPy2000 module types. It puts in place a flexible framework for presenting stimuli and conducting experiments. Your implementation of an application module would begin as follows: 

class BciApplication (BciGenericApplication):

and would go on to implement methods called Transition, Phases, Frame and Process in order to specify the application module's behaviour.
