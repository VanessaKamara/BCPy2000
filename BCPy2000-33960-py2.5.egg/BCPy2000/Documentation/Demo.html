Demo

There are three demo setups. The python source files used to implement their behaviour, and the sound files used for feedback, are found in the python directory.

Contents

  1. The Techno Triangle
  2. The Same Again, Please
  3. No Bells or Whistles

The Techno Triangle

The first demo is launched by double-clicking on PythonDemo1_Triangle.bat in the batch directory. Edit this file and take a look at the contents: it starts PythonSource.exe with a command-line option specifying TrefoilSource.py as the signal source module implementation, PythonSignalProcessing.exe with no developer file (simple pass-through processing), and PythonApplication.exe with TriangleApplication.py as the application module implementation. It also ensures that the operator loads the necessary parameter file automatically: you should be able to press "Set Config" immediately (although there may be a minute's delay for compilation of various things if you've never used the system before) and then "Start".

TriangleApplication.py is a toy BCI experiment in which the subject imagines moving left hand, right hand or feet in order to guide the cursor into one of the three corners of the triangle. At the same time, depending on the parameter settings of the PythonApp tab, the subject receives colour feedback and sound feedback - each imagination task is associated with a colour and/or a continuous sound, and these get brighter/louder as the probability of the associated class increases.

The application module expects an input signal of 3 or 4 channels, the value of which should be the estimated probability of each of the classes (the optional 4th channel is the probability of the zero class or "rest" class). In fact, in the demo, no such sensible values are delivered: TrefoilSource.py just generates an artificial 3-channel signal that makes the cursor visit each of the corners in turn, and the signal-processing module passes this through unchanged.

The triangle demo showcases a number of the optional higher-level tools - note, for example, the "state monitors" that show you the value of certain state variables. These are turned off (according to the logic implemented in TriangleApplication.py's Initialize hook) if you also turn off the packet clock in the bottom right corner, by unchecking the ShowSignalTime parameter.

Note the interactivity of the modules. For example, in the PythonSig shell, type the following:
self.Process = lambda x: x.A**2
and notice that the cursor now goes deeper into the corners due to the new "signal processing".

The Same Again, Please

The second demo is launched by opening PythonDemo2_Playback.bat. Here we see the same application, but the signal and experimental sequence are in fact played back from the first file you recorded using demo 1 (or from whatever file you specify in the PlaybackFileName parameter of the Source tab). See the documentation on playback mode for more details.

No Bells or Whistles

The third demo is launched from PythonDemo3_Template.bat. Here, very little happens. No parameter file is loaded, the default developer file names, Bci*.py, are used, and the implementations in these files are very minimal: didactically helpful, we hope, but experimentally meaningless.
