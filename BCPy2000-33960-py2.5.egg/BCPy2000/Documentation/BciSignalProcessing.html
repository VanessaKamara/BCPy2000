BciSignalProcessing

BciSignalProcessing is the name of a class, implemented by you in one of the developer files. The filename, by default, is also BciSignalProcessing.py but this can be changed by specifying the command-line option --PythonSigClassFile to the signal processing binary.

The BciSignalProcessing class is instantiated when the BCI2000 starts, and if the signal-processing module's shell is running, you can address this instance from the shell prompt as self.

Your BciSignalProcessing must be a subclass of BciGenericSignalProcessing, which is defined in the framework module BCPy2000.GenericSignalProcessing. Your implementation should not define __init__ and __del__ methods, but rather use the appropriate hooks named according to the BCI2000 conventions.

The complete implementation of a signal-processing module which outputs the root-mean-square of the input signals (computed, rather crudely, on a per-packet basis) would be:

class BciSignalProcessing (BciGenericSignalProcessing):

    def Preflight(self, in_signal_props):
        nchannels = self.in_signal_dim[0]
        # specify an output packet size of 1 sample
        return (nchannels, 1)

    def Process(self, x):
        return (x.A**2.0).mean(axis=1)**0.5
