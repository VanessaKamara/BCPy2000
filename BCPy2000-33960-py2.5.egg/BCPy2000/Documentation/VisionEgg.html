VisionEgg

The Vision Egg is a high-level python framework written by Andrew Straw. It is aimed at experimental psychologists and vision scientists, and its main purpose is the presentation and control of visual stimuli. It is described in:

Straw, A. D. (2008)
VisionEgg: An Open-Source Library for Realtime Visual Stimulus Generation.
Frontiers in Neuroinformatics 2:4. doi: 10.3389/neuro.11.004.2008

By default, BCPy2000 visual stimuli are expected to be VisionEgg Stimulus objects, so you may wish to familiarize yourself with it at the official site http://visionegg.org and bookmark the reference manual.

The BCPy2000 Application module automatically handles the updating and redrawing of stimuli. All you need to do is:

  1. Optionally, during your Preflight hook, set any VisionEgg.config parameters you want in order to specify window behaviour before VisionEgg opens (init screen also provides an interface to the most common parameters).
  2. During your Initialize hook, register your stimuli with calls to self.stimulus. For example:

    from VisionEgg.Text import Text
    ...
    t = Text('Hello World', position=(100,100)))
    self.stimulus('foo', t)

    Alternatively, instead of an instance, you can pass the class and parameters, for deferred construction:

    self.stimulus('blah', Text, text='Hello Again World', position=(100,50))

  3. During your Process, Transition, or Frame hooks, update the stimulus parameters to create the behaviour you want. For example:

    self.stimuli['blah'].color=(1,0,0)

    will turn the stimulus you have registered as 'blah' red.

Once the VisionEgg window has been opened (following "Set Config") the current VisionEgg.Core.Screen object is available at self.screen._screen and the current VisionEgg.Core.Viewport is at self.screen._viewport, but it is not usually necessary to access these explicitly. To change the background colour or query the screen size you can interact with the self.screen object directly:

self.screen.bgcolor = (0,0,0)
width,height = self.screen.size

The self.screen object is of class VisionEggRenderer but, you can also write your own renderer if you want an alternative to VisionEgg.
