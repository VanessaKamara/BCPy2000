Signal Source Binary

The signal source binary resides in the BCI2000 installation directory and is called PythonSource.exe.

The abbreviation "PythonSrc" is used as a prefix for all the module's command-line options, and as the name of the main BCI2000 ADC filter registered by the module.

The signal source binary does the following:

  1. starts an embedded python engine
  2. opens a console window (unless the command-line options specify a log file and explicitly disable the shell)
  3. imports symbols from the framework module BCPy2000.GenericSource
  4. changes to the desired working directory and adds it to python's sys.path
  5. runs the developer's signal-generation file (by default, called BciSource.py)
  6. creates an instance of the developer-defined class BciSource (referred to as self)
  7. starts an interactive IPython shell if desired
  8. acts as an intermediary between self's methods and (via TCP/IP connections) the other modules of the BCI2000.
