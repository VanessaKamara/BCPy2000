Run

Definition: run

Normally this is synonymous with block, meaning a sequence of trials performed one after the other without a break. However, BCPy2000 provides for the possibility that multiple blocks (according to our definition of that term) are performed and saved continuously, together with the interblock breaks, in the same file. According to BCI2000 terminology the file corresponds to a "run", so then there are multiple BlocksPerRun, as dictated by the parameter of that name.

BCI2000 data file names usually end in SsssRrr.dat where sss denotes the session number and rr denotes the run number within the session.
