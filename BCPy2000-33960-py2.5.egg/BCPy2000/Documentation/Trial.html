Trial

Definition: trial

A period of continuous concentration during which the subject makes one attempt to transmit one particular symbol out of a given set (maybe one attempt to imagine left hand movement instead of right, maybe an attempt to transmit a particular letter in a grid-speller) or achieve a certain short-term goal (like guiding the cursor to a target). Usually a trial lasts several seconds, and may (as in the last example) be of variable length even in the same experiment. A trial might consist of a number of consecutive subtrials (particularly in a grid-speller experiment). Often, a trial is followed by a short pause called the inter-trial interval, which is a few seconds (preferably a somewhat randomized duration) in which the subject can blink/swallow/shift around/forget what they are supposed to be doing, before the next trial begins.

There is no necessity to break your experiment into trials, but it is very common. Therefore, if you want to do so, the BCPy2000 application module automatically implements a CurrentTrial state which counts up to the number specified by the TrialsPerBlock parameter. The presentation phase on which CurrentTrial is incremented, and what to do when a block is over, are specified by the self.design API method.
